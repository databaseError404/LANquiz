
`LAN Quiz` — локальный веб-сервер для проведения викторины в одной сети.

Программа работает через браузер, отдельные приложения игрокам устанавливать не нужно.

- Ведущий запускает веб-сервер на компьютере под управлением Windows: файл `quiz-host.exe`.

![Скриншот 1](doc/Screenshot_1.jpg)

- После запуска у ведущего автоматически открывается host-экран: `http://localhost:8080/host`.

![Скриншот 2](doc/Screenshot_2.jpg)

- Также доступен экран `/screen` со статистикой, например `http://192.168.88.127:8080/screen`. Его можно открыть у ведущего и вывести через проектор.

![Скриншот 3](doc/Screenshot_3.jpg)

- Ведущий может показать QR-код для подключения или сообщить адрес вручную.
- Все участники должны быть подключены к одной Wi‑Fi-сети.
- Участники (команды) подключаются со своих телефонов по указанному адресу и вводят название команды.
- По мере подключения у ведущего отображаются подключённые команды, сортировка — по алфавиту.
- Ведущий может вручную добавить команду прямо на своей странице (например, если команда временно без телефона).
- Для команд без смартфонов можно использовать RF-пульты (433 МГц) через USB-приёмник, подключённый как COM-порт.
- Ведущий может удалить или переименовать команду прямо в списке.
- После переименования у ведущего новое название автоматически обновляется и у соответствующего игрока.
- Если команда удалена ведущим, у игрока автоматически снова открывается форма ввода имени команды.
- Ведущий управляет вопросами на странице ведущего.
- Для начала вопроса ведущему достаточно выбрать правильный ответ — запустится таймер.
- Участники видят, что вопрос начался, и выбирают ответ: **А / Б / В / Г**.

![Скриншот 4](doc/Screenshot_4.jpg)

- Ведущий может остановить вопрос кнопкой «Остановить» (приём ответов прекращается, но правильный ответ игрокам ещё не показывается).
- Вопрос завершается кнопкой «Завершить» (после этого игрокам открывается правильный ответ).
- У ведущего в столбце «Ответ» доступен выпадающий список `— / А / Б / В / Г` для каждой команды: можно назначить, изменить или очистить ответ (очистка — выбор `—`) до завершения вопроса.
- В режиме «Принять оставшиеся ответы» (late) ведущий также может менять ответы команд вручную через этот список.
- Если у вопроса запущен таймер, то до `00:00` ничего не происходит автоматически, даже если уже ответили все команды (ответ можно менять до конца времени).
- В момент `00:00`:
  - если ответили все зарегистрированные команды, выполняется поведение как у кнопки «Завершить» (правильный ответ раскрывается),
  - если ответили не все, выполняется поведение как у кнопки «Остановить» (правильный ответ не раскрывается, пока ведущий не нажмёт «Завершить»).
- Если некоторые участники не ответили (например, ждут подсказку), их ответы можно принять позже кнопкой «Принять оставшиеся ответы».
- Когда вопрос завершён, у участников показывается их вариант ответа и правильный вариант.
- Если команда привязана к RF-пульту, она также может отвечать во время приёма ответов кнопками пульта.
- После завершения текущего вопроса кнопкой «Следующий вопрос» можно перейти к следующему.
- Кнопка «Следующий вопрос» активируется только когда текущий вопрос завершён и ответ зафиксирован.
- Если в этот момент срабатывает привязанный RF-пульт ведущего (кнопка **А/Б/В/Г**), это обрабатывается как «Следующий вопрос», затем сразу как выбор правильного ответа нового вопроса, и таймер нового вопроса запускается автоматически.
- У ведущего отображается статистика по всем командам, у каждого игрока — только своя.
- Если нужно повторно провести текущий вопрос, используйте кнопку **«Переиграть вопрос»** (в дополнительных действиях).
- Текущее состояние игры автоматически сохраняется в файл `data/state.json` (команды, текущий вопрос, ответы и история).
- Начать новую серию вопросов можно кнопкой «Сброс статистики и новый раунд» в блоке «Дополнительные действия».
- При сбросе статистики предыдущие данные сохраняются в CSV-файл в папке `data/stats/`.
- На экране `.../screen` показывается только статистика, а QR — только если включена соответствующая галочка у ведущего.

## Режим «Раунд с несгораемыми суммами»

- На экране ведущего рядом с «Можно менять ответ» есть галочка **«Раунд с несгораемыми суммами»**.
- В этом режиме стоимость правильного ответа считается не как `+1`, а по таблице прогрессии.
- Для каждой команды в столбце «Действие» можно задать до **3 несгораемых сумм** (редактирование доступно только когда режим включён).
- Если команда отвечает правильно — её счёт переходит на **следующее** значение таблицы.
- Если команда отвечает неправильно — её счёт откатывается к ближайшей несгораемой сумме (не выше текущего счёта) или к `0`, если подходящей несгораемой суммы нет.
- Прогресс по стоимости считается **отдельно для каждой команды**.
- На экране ведущего под названием команды (в этом режиме) показывается подсказка с её несгораемыми суммами.
- На экране игрока показываются:
  - несгораемые суммы команды,
  - текущий счёт (стоимость следующего вопроса показывается в блоке счёта).

## Подсветка ответов у игрока

- До завершения вопроса выбранный игроком вариант подсвечивается синим.
- После завершения:
  - правильный вариант подсвечивается зелёным,
  - выбранный неверный вариант — красным,
  - если игрок не отвечал, правильный вариант показывается серым.

## Таблица прогрессии стоимости

- Таблица стоимости хранится в текстовом файле [`data/score-progression.txt`](data/score-progression.txt).
- Формат: по одному числу в строке, значения должны быть неотрицательные и строго возрастающие.
- Пример стандартной таблицы:

```text
0
100
200
300
500
700
1000
2000
3000
5000
10000
15000
20000
30000
```

- Если файла нет, он создаётся автоматически с таблицей по умолчанию.

## RF-пульты (COM-порт)

- На экране ведущего в блоке **RF пульты** выберите COM-порт приёмника и нажмите кнопку **Подключить RF / Отключить RF** (это одна и та же переключаемая кнопка).
- Кнопка **Обновить** перечитывает список COM-портов.
- Кнопка **Добавить команду с RF пультом** включает/выключает режим привязки.
- Кнопка **Привязать RF пульт ведущего** включает/выключает отдельный режим привязки пульта ведущего.
- В режиме привязки нажмите на нужном пульте кнопку **А** — создастся команда с именем вида `RF XXXXX` (где `XXXXX` — ID пульта, ведущий затем может переименовать).
- После успешного добавления одной команды режим привязки автоматически выключается.
- Если в режиме привязки нажать кнопку **А** на уже привязанном пульте, в строке статуса показывается подсказка, к какой команде он уже привязан.
- Режим привязки автоматически выключается по таймауту `30` секунд.
- В режиме привязки пульта ведущего нажмите кнопку **А** на нужном пульте — пульт ведущего привяжется по базовому ID, режим привязки ведущего автоматически выключится.
- Если выбранный для ведущего пульт уже привязан к команде, привязка к ведущему отклоняется с предупреждением в статусе RF.
- При удалении команды её RF-привязки очищаются, чтобы этот пульт можно было затем привязать ведущему.
- Привязка выполняется по «базовому» ID пульта (вся посылка без последнего **полубайта**).
- Во время игры ответы принимаются по последнему полубайту посылки:
  - `8` → **А**
  - `4` → **Б**
  - `2` → **В**
  - `1` → **Г**
- Если срабатывает привязанный пульт ведущего, кнопки **А/Б/В/Г** дублируют выбор правильного ответа у ведущего (как кнопки на странице ведущего), включая автостарт вопроса, когда это разрешено логикой текущего состояния.
- Для автостарта через RF-пульт ведущего используется текущая длительность вопроса с host-экрана; таймер запускается на это значение.
- Для RF-пультов повторное нажатие уже выбранного варианта не снимает ответ; смена ответа происходит только при выборе другого варианта.
- Для строк приёмника используются метки удержания:
  - `=>` — начало нажатия
  - `==` — удержание
  - `<=` — отпускание
- В обработку действия идут события `=>` (начало нажатия), чтобы избежать повторов от удержания.
- В статусе RF длинные сообщения переносятся на несколько строк; техническое сообщение о закрытии порта показывается в сокращённом виде (`COM порт закрыт`).
- Выбранный COM-порт и привязки RF-пультов сохраняются в `data/state.json`.
- При старте сервера выполняется авто-переподключение к последнему выбранному COM-порту. Если порт недоступен, сервер продолжает работу, а ошибка показывается в статусе RF у ведущего.

## Установка Go и сборка проекта

### 1. Установить Go через `winget`

В `cmd.exe` выполните команду:

```bat
winget install -e --id GoLang.Go
```

После установки закройте терминал и откройте новый, затем проверьте:

```bat
go version
```

### 2. Скачать зависимости и собрать проект

В папке проекта выполните:

```bat
go mod download
go build -o quiz-host .
```

Либо используйте готовый файл [`build.bat`](build.bat):

```bat
build.bat
```

После сборки будет создан файл [`quiz-host.exe`](quiz-host.exe).

### 3. Запуск проекта

Запуск без предварительной сборки:

```bat
go run .
```

или через готовый файл [`run.bat`](run.bat):

```bat
run.bat
```

Запуск уже собранного файла:

```bat
quiz-host.exe
```

После запуска у ведущего автоматически откроется страница `http://localhost:8080/host`.
