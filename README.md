
`LAN Quiz` — локальный веб-сервер для проведения викторины в одной сети.

Программа работает через браузер, отдельные приложения игрокам устанавливать не нужно.

- Ведущий запускает веб-сервер на компьютере под управлением Windows: файл `quiz-host.exe`.

![Скриншот 1](doc/Screenshot_1.jpg)

- После запуска у ведущего автоматически открывается host-экран: `http://localhost:8080/host`.

![Скриншот 2](doc/Screenshot_2.jpg)

- Также доступен экран `/screen` со статистикой, например `http://192.168.88.127:8080/screen`. Его можно открыть у ведущего и вывести через проектор.

![Скриншот 3](doc/Screenshot_3.jpg)

- Ведущий может показать QR-код для подключения или сообщить адрес вручную.
- Все участники должны быть подключены к одной Wi‑Fi-сети.
- Участники (команды) подключаются со своих телефонов по указанному адресу и вводят название команды.
- По мере подключения у ведущего отображаются подключённые команды, сортировка — по алфавиту.
- Ведущий может не только удалить команду, но и переименовать её прямо в списке.
- После переименования у ведущего новое название автоматически обновляется и у соответствующего игрока.
- Если команда удалена ведущим, у игрока автоматически снова открывается форма ввода имени команды.
- Ведущий управляет вопросами на странице ведущего.
- Для начала вопроса ведущему достаточно выбрать правильный ответ — запустится таймер.
- Участники видят, что вопрос начался, и выбирают ответ: **А / Б / В / Г**.

![Скриншот 4](doc/Screenshot_4.jpg)

- Ведущий может остановить вопрос кнопкой «Остановить» (приём ответов прекращается, но правильный ответ игрокам ещё не показывается).
- Вопрос завершается кнопкой «Завершить» (после этого игрокам открывается правильный ответ).
- Если некоторые участники не ответили (например, ждут подсказку), их ответы можно принять позже кнопкой «Принять оставшиеся ответы».
- Когда вопрос завершён, у участников показывается их вариант ответа и правильный вариант.
- После завершения текущего вопроса кнопкой «Следующий вопрос» можно перейти к следующему.
- Кнопка «Следующий вопрос» активируется только когда текущий вопрос завершён и ответ зафиксирован.
- У ведущего отображается статистика по всем командам, у каждого игрока — только своя.
- Если нужно повторно провести текущий вопрос, используйте кнопку **«Переиграть вопрос»** (в дополнительных действиях).
- Текущее состояние игры автоматически сохраняется в файл `data/state.json` (команды, текущий вопрос, ответы и история).
- Начать новую серию вопросов можно кнопкой «Сброс статистики и новый раунд» в блоке «Дополнительные действия».
- При сбросе статистики предыдущие данные сохраняются в CSV-файл в папке `data/stats/`.
- На экране `.../screen` показывается только статистика, а QR — только если включена соответствующая галочка у ведущего.

## Режим «Раунд с несгораемыми суммами»

- На экране ведущего рядом с «Можно менять ответ» есть галочка **«Раунд с несгораемыми суммами»**.
- В этом режиме стоимость правильного ответа считается не как `+1`, а по таблице прогрессии.
- Для каждой команды в столбце «Действие» можно задать до **3 несгораемых сумм** (редактирование доступно только когда режим включён).
- Если команда отвечает правильно — её счёт переходит на **следующее** значение таблицы.
- Если команда отвечает неправильно — её счёт откатывается к ближайшей несгораемой сумме (не выше текущего счёта) или к `0`, если подходящей несгораемой суммы нет.
- Прогресс по стоимости считается **отдельно для каждой команды**.
- На экране ведущего под названием команды (в этом режиме) показывается подсказка с её несгораемыми суммами.
- На экране игрока показываются:
  - несгораемые суммы команды,
  - текущий счёт (стоимость следующего вопроса показывается в блоке счёта).

## Подсветка ответов у игрока

- До завершения вопроса выбранный игроком вариант подсвечивается синим.
- После завершения:
  - правильный вариант подсвечивается зелёным,
  - выбранный неверный вариант — красным,
  - если игрок не отвечал, правильный вариант показывается серым.

## Таблица прогрессии стоимости

- Таблица стоимости хранится в текстовом файле [`data/score-progression.txt`](data/score-progression.txt).
- Формат: по одному числу в строке, значения должны быть неотрицательные и строго возрастающие.
- Пример стандартной таблицы:

```text
0
100
200
300
500
700
1000
2000
3000
5000
10000
15000
20000
30000
```

- Если файла нет, он создаётся автоматически с таблицей по умолчанию.
